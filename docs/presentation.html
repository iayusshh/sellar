🔐 Security & Architecture

Built Secure by Default

Security is enforced at the database layer — not just the client. Every API call, every row, every wallet transaction is protected.

🛡️

Row-Level Security (RLS)

PostgreSQL RLS policies on every table. Creators can only read/write their own products, wallets, and transactions — regardless of client state.

⚡

Atomic Commission Split

Payment settlement is a single PostgreSQL stored procedure. Commission deduction and creator wallet credit happen atomically — no race conditions possible.

🔑

JWT Authentication

Supabase Auth issues signed JWTs for every session. Edge Functions verify tokens server-side before processing any payment operations.

🔗

Secure Join Tokens

Webinar join URLs are generated server-side via Edge Functions. Only verified purchasers within the join window receive valid tokens.

📡

Webhook Verification

Cashfree webhook signature validation prevents spoofed payment notifications. Every webhook is cryptographically verified before processing.

🏦

RBI-Compliant Payments

All payments flow through Cashfree — an RBI-licensed payment aggregator. UPI, net banking, and card payments are fully compliant.

⚙️ Technology

Modern Cloud-Native Stack

Built on proven, production-grade technologies. No custom backend required — Supabase handles auth, database, storage, and serverless functions.
